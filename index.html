Day3SAME DAY ▲
Draw Your Schematic
Logical wiring diagram only — not physical layout yet. Connect every component to the MCU.
  • Place all symbols: MCU, 6 switches, 6 diodes, 2 encoders, 6 RGB LEDs
  • Wire switch matrix: switch → diode (cathode toward column) → column pin
  • Wire RGB daisy chain: MCU GP → LED1 DIN → LED1 DOUT → LED2 DIN...
  • Wire encoder A, B, SW pins to free GPIO pins
  • Run ERC (Electrical Rules Check) — fix every error before proceeding
Resources
🔌 QMK Wiring Reference
DAYS 4–6 🔌 PCB Layout & Order Phase 2
Day4SAME DAY ▲
Physical PCB Placement
Import footprints from schematic and place them physically on your board.
  • Open PCB editor → "Update PCB from Schematic" to import all footprints
  • Draw board outline on Edge.Cuts layer (~90mm × 72mm for this layout)
  • Place 6 MX switch footprints in 2×3 grid (19.05mm standard MX spacing)
  • Place 2 EC11 encoder footprints above the key grid
  • Place SK6812 LEDs under each switch (south-facing orientation)
  • Place MCU and USB-C socket — USB-C must reach the board edge
Day5SAME DAY ▲
Route Traces & Ground Pour
Connect everything with copper traces. Add a ground pour for a cleaner, pro result.
  • Route switch matrix: columns on F.Cu, rows on B.Cu using vias to cross
  • Route RGB data chain between LEDs (short traces, same layer if possible)
  • Use 0.5mm traces for power (VCC/GND), 0.25mm for signal lines
  • Add GND copper pour on both layers (Edit → Fill All Zones)
  • Run DRC (Design Rules Check) — zero errors before export!
Day6SAME DAY ▲
Export Gerbers & Order! 🎉
Add silkscreen, export Gerbers, and place your order. Boards arrive in ~7 days!
  • Add silkscreen labels: SW1–SW6, ENC1, ENC2, board name on F.Silkscreen layer
  • File → Plot → Select all layers → Click Plot (generates Gerber files)
  • Generate Drill Files in Excellon format into same folder
  • ZIP the Gerber folder → upload to JLCPCB → order 5 boards ($5–10)
  • Order all components from AliExpress/LCSC while you're at it

PCB Arrives — Inspect & Solder SMD

Day11SAME DAY ▲
Solder Switches, Encoders & MCU
The satisfying step — your board really starts to look like a macropad!
  • Snap all 6 switches into the plate, set plate+switches onto PCB
  • Tack corner switches first to lock alignment, then solder all 12 pins
  • Solder both EC11 encoders — 5 signal pins + 2 mounting legs each
  • Solder Raspberry Pi Pico via pin headers or direct mount

Day12SAME DAY ▲
First Power-On Test
Check for shorts with a multimeter first — then plug it in and watch those LEDs glow!
  • Multimeter in continuity mode: confirm no short between VCC and GND pins
  • Plug into USB — MCU should enumerate as USB device (check Device Manager)
  • Flash basic test firmware → verify all 6 LEDs light up in sequence
  • Reflow any cold joints: add flux, briefly reheat with iron tip

⚠️ Soldering Order Matters! Always go smallest to tallest: SMD diodes → SK6812 LEDs → switches → encoders → MCU. Getting this wrong makes later components very hard to reach.

Day13SAME DAY ▲
Write Your QMK Firmware
Configure your keyboard brain — pins, keys, RGB effects, and encoder actions.

  • Edit config.h: MATRIX_ROW/COL_PINS, WS2812_DI_PIN, ENCODERS_PAD_A/B
  • Edit keymap.c: assign 6 keycodes (git commit, save, build, run, format, push)
  • Add encoder_update_user() callback: knob 1 = scroll, knob 2 = volume
  • Enable in rules.mk: RGB_MATRIX_ENABLE = yes, ENCODER_ENABLE = yes
  • Build: qmk compile -kb mymacropad -km default — fix any errors

🔌 Deep Dive

PCB Design Explained

1
Switch Matrix — Why diodes?
For 6 keys, use a 2×3 matrix — 2 rows × 3 columns = 5 MCU pins instead of 6. Diodes prevent "ghosting" (phantom keypresses) when multiple keys are held. Rule: Row → switch → diode → Column. Diode cathode (stripe side) always points toward the column wire.
2
RGB LED Wiring (Addressable LEDs)
SK6812 and WS2812B LEDs need only one data wire from your MCU! They daisy-chain: MCU GPIO → LED1 DIN → LED1 DOUT → LED2 DIN →.... Add a 300Ω resistor on the MCU data line and a 100nF bypass cap between VCC and GND near each LED to prevent flickering.
3
EC11 Encoder Pinout
EC11 has 5 pins: A, B, C (GND), SW1, SW2. Connect A and B to MCU GPIOs (pull-ups enabled in firmware). Connect C to GND. SW1+SW2 are the push-click switch — wire like a regular key switch. Enable ENCODER_ENABLE = yes in QMK rules.mk.
4
RP2040 Pin Recommendation
Suggested: GP0–GP1 = matrix rows, GP2–GP4 = matrix columns, GP5 = RGB data, GP6–GP7 = encoder 1 (A, B), GP8–GP9 = encoder 2 (A, B), GP10–GP11 = encoder push buttons. Write this down — it goes directly into config.h!
💻 Code Reference

QMK Firmware Snippets

config.h — Pin & Feature Configuration
// config.h #pragma once #include "config_common.h" // Matrix: 2 rows × 3 cols = 6 keys on 5 pins #define MATRIX_ROWS 2 #define MATRIX_COLS 3 #define MATRIX_ROW_PINS { GP0, GP1 } #define MATRIX_COL_PINS { GP2, GP3, GP4 } #define DIODE_DIRECTION COL2ROW // RGB LEDs #define WS2812_DI_PIN GP5 #define RGB_MATRIX_LED_COUNT 6 #define RGB_MATRIX_MAXIMUM_BRIGHTNESS 200 // Rotary Encoders #define ENCODERS_PAD_A { GP6, GP8 } #define ENCODERS_PAD_B { GP7, GP9 } #define ENCODER_RESOLUTION 4
keymap.c — Keys + Encoder Callbacks
// keymap.c #include QMK_KEYBOARD_H const uint16_t PROGMEM keymaps[][MATRIX_ROWS][MATRIX_COLS] = { [0] = LAYOUT( // Row 1: Format, Save, Toggle Sidebar LSFT(LALT(KC_F)), LCTL(KC_S), LCTL(KC_B), // Row 2: Build, Run, Custom macro (git push etc) LCTL(LSFT(KC_B)), KC_F5, KC_F13 ) }; // Encoder: Knob 1 = scroll, Knob 2 = volume bool encoder_update_user(uint8_t index, bool clockwise) { if (index == 0) { clockwise ? tap_code(KC_MS_WH_DOWN) : tap_code(KC_MS_WH_UP); } else { clockwise ? tap_code(KC_VOLU) : tap_code(KC_VOLD); } return false; }

3D Printed Sandwich
Two-piece: bottom shell + switch plate. 2–4 hours to print in PLA or PETG. Most accessible option if you have a printer or a nearby makerspace.
✅ Best for this Build

Upload your STL with your PCB order. They print and ship together. Resin gives incredible detail. No printer needed at all.
